https://enrollx-c17aa.web.app/admin-login.html

The Admin Panel is the control center of EnrollX. Super Admins have unrestricted access. Department, Placement, and Exam Admins have scoped access based on their role type.

  • View system-wide statistics dashboard

Admin Roles Explained

Role Access Scope Restrictions
Super Admin Everything — all departments, all features None. Can edit/delete all admins, courses, faculty.
Department Admin Only their assigned department's students, courses, faculty, enrollments Cannot access Administrators page. Data is filtered by department.
Placement Admin Student data and placement-relevant info Cannot access Courses or Faculty pages. Cannot add/delete data.
Exam Admin Results and exam-related data Cannot add/delete records. No access to add buttons.

5. User Workflows & Action Flows

Step-by-step process for every major action in the system

Student Registration & First Login

  1. Visit Student Login Page

    Go to enrollx-c17aa.web.app/login.html — click "Register" to create a new account.

  2. Fill Registration Form

    Enter: Full Name, Student ID (e.g. CS2023001), Email, Password, Department, Semester, Phone number.

  3. Account Created

    Firebase Auth creates the login account. A Firestore document is created in the students collection with all profile data.

  4. Redirect to Dashboard

    Student is automatically logged in and redirected to their dashboard.

  5. Browse & Enroll in Courses

    Navigate to "My Courses" → find a course → click "Enroll". Available seats decrease by 1.

Faculty Account Creation by Admin

  1. Super Admin logs into Admin Panel

    Using /admin-login.html with admin credentials.

  2. Navigate to Faculty → Add Faculty

    Fill in: Name, Email, Password (min 6 chars), Department, Designation, Phone.

  3. Firebase Account Created

    A Firebase Auth account is created for the faculty. A Firestore document is saved in the faculty collection with role = "faculty".

  4. ⚠️ Admin is Signed Out (Browser Limitation)

    Creating a new user via browser Firebase SDK signs out the current session. Admin must log back in at /admin-login.html.

  5. Assign Course to Faculty

    Go to Courses → Edit the course → select the faculty from the dropdown → Save. The faculty can now see this course in their portal.

Password Reset Flow

📧 Via Email

  1. Go to Forgot Password

    Click "Forgot Password?" on login page.

  2. Enter Email

    Type registered email address.

  3. Check Inbox/Spam

    Firebase sends a reset link. Check spam/junk folder.

  4. Click Link → Set New Password

    Reset completes via Firebase.

📱 Via Phone Number

  1. Click "Phone Number" Tab

    On the forgot password page.

  2. Enter Registered Phone

    System looks up the account by phone number.

  3. Email Shown (Masked)

    Shows har***@gmail.com for privacy.

  4. Send Reset Link

    Firebase sends reset email to that account.

7. Results & Grading System

How marks, grades, and CGPA are calculated automatically

📝

Internal Marks

Faculty enters a score from 0 to 50. This typically covers assignments, mid-term tests, attendance, and class performance.

📋

External Marks

Faculty enters a score from 0 to 50. This represents the end-semester or final examination score.

Grading Scale

Total Marks (out of 100) Grade Grade Points Performance Level
90 – 100 O 10.0 Outstanding
80 – 89 A+ 9.0 Excellent
70 – 79 A 8.0 Very Good
60 – 69 B+ 7.0 Good
50 – 59 B 6.0 Above Average
40 – 49 C 5.0 Satisfactory
Below 40 F 0.0 Fail

CGPA Calculation Formula

🧮

Formula

CGPA = Σ (Grade Points × Credits) ÷ Σ Credits
For each course: multiply the grade point by the course's credit hours. Sum all, then divide by total credits. E.g., two 4-credit courses with 8.0 and 9.0 grade points → CGPA = (32 + 36) / 8 = 8.5

What Students See in Results Page

Column Description
Course Name Name of the enrolled course
Course Code Unique identifier (e.g. CS401)
Internal (50) Marks entered by faculty for internal assessment
External (50) Marks entered by faculty for end-semester exam
Total (100) Internal + External, auto-calculated
Grade Letter grade based on total (O/A+/A/B+/B/C/F)
CGPA Shown at bottom of page as weighted average across all results

8. Security & Authentication

How data is protected and access is controlled

Authentication System

  1. Firebase Authentication

    All users (students, faculty, admins) log in using Firebase Auth — email + password. Firebase handles password hashing and token management securely.

  2. Role Verification on Login

    After logging in, the system checks the Firestore collection corresponding to the portal. Student login checks students collection, Faculty checks faculty, Admin checks admins. If the document doesn't exist, access is denied even with valid credentials.

  3. Route Guards (Page Protection)

    Every protected page calls requireAuth(role) on load. If the user is not authenticated or doesn't have the correct role, they are redirected to the appropriate login page.

  4. Firestore Security Rules

    Server-side rules restrict database operations. Students can only update availableSeats on courses (for enrollment/drop). Admins can write to any collection. Unauthenticated users have no database access.

Firestore Data Collections

Collection Documents Stored Key Fields
students One per student (doc ID = Firebase Auth UID) name, email, studentId, department, semester, phone, role
faculty One per faculty (doc ID = Firebase Auth UID) name, email, department, designation, phone, role
admins One per admin (doc ID = Firebase Auth UID) name, email, adminType, adminDepartment, role
courses One per course courseName, courseCode, credits, department, semester, facultyId, facultyName, totalSeats, availableSeats
enrollments One per student-course pair studentId, courseId, courseName, status (enrolled/dropped), enrolledAt
results One per student-course pair studentId, courseId, facultyId, internalMarks, externalMarks, total, grade, gpaPoints, createdAt
⚠️ Important: A Firebase Auth account alone is not enough for access. Every user must also have a matching Firestore document in their respective collection (students / faculty / admins). If a document is missing, login will be rejected with "Access denied".
